### A
*   **Algoritme**: Een vaste set regels of stappen die een programma volgt om een specifiek resultaat te bereiken (bijv. sorteren op datum).
*   **API (Application Programming Interface)**: Een manier voor verschillende software-systemen om met elkaar te praten.

### B
*   **Back-end**: De "achterkant" van de website (PHP & Database) waar de berekeningen en opslag plaatsvinden.
*   **BCRYPT**: Een zeer veilig algoritme om wachtwoorden te hashen. Het is ontworpen om traag te zijn, waardoor het hacken via 'brute force' bijna onmogelijk wordt.

### C
*   **Client-side**: Alles wat in de browser van de gebruiker gebeurt (HTML, CSS, JavaScript).
*   **CSRF (Cross-Site Request Forgery)**: Een aanval waarbij een hacker acties uitvoert namens een ingelogde gebruiker. Wij voorkomen dit door acties alleen via POST-verzoeken toe te staan.

### D
*   **Data-integriteit**: De mate waarin data in de database correct, compleet en logisch is.
*   **DRY (Don't Repeat Yourself)**: Een basisregel voor programmeurs: schrijf nooit twee keer dezelfde code, maar maak er een herbruikbare functie van.

### F
*   **Front-end**: De visuele kant van de website waar de gebruiker mee interacteert.

### G
*   **Glassmorphism**: Een moderne ontwerp-stijl die gebruik maakt van transparantie en achtergrondvervaging om een glas-achtig effect te creÃ«ren.

### H
*   **Hashing**: Het omzetten van een tekst (zoals een wachtwoord) in een unieke code die je niet meer terug kunt draaien.

### N
*   **Normalisatie**: Het proces waarbij een database zo wordt ingericht dat er geen dubbele data ontstaat (zoals onze `Games` tabel).

### O
*   **OWASP**: De 'Open Web Application Security Project'. Dit is de wereldwijde standaard voor het veilig bouwen van web-applicaties.

### P
*   **PDO (PHP Data Objects)**: Een krachtige en veilige interface in PHP om met verschillende soorten databases te communiceren.
*   **Prepared Statements**: Een techniek die SQL-commando's scheidt van gebruikersdata om SQL-injectie te voorkomen.

### S
*   **Server-side**: Alles wat op de server van de aanbieder gebeurt (PHP).
*   **Soft Delete**: Een record niet echt verwijderen, maar op 'onzichtbaar' zetten via een tijdstempel.
