# 1. Architecturale Retrospectief

Toen ik begon aan de GamePlan Scheduler, was mijn grootste angst "Spaghetti Code". Door de jaren heen heb ik geleerd dat complexiteit de vijand is van stabiliteit. In dit project heb ik de **"Logic Centralization Strategy"** toegepast:
- **Waarom PDO?**: Aanvankelijk overwoog ik MySQLi, maar de object-georiÃ«nteerde aard van PDO sluit beter aan bij de groei-strategie van het project. Het stelt ons in staat om later makkelijk over te stappen naar PostgreSQL of SQL Server zonder de hele code her te schrijven.
- **De Glue Theory**: Mijn `functions.php` werkt als de lijm tussen de interface en de data. Door alle herbruikbare logica hier te concentreren, heb ik een DRY (Don't Repeat Yourself) score behaald die ver boven het gemiddelde ligt.

# 7. Ethiek & Verantwoordelijkheid

Als developer ben ik verantwoordelijk voor de data van mijn gebruikers. In de GamePlan Scheduler heb ik ethiek vertaald naar techniek:
- **Data Minimalisatie**: We slaan alleen op wat strikt noodzakelijk is. Geen onnodige tracking, geen verkoop van data.
- **Transparantie**: De privacy-pagina legt in klare taal uit wat er met de data gebeurt. Dit is mijn invulling van de AVG (GDPR) principes op een professioneel niveau.
